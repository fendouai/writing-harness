# Module Specs

Use this when you need a module-by-module writing plan.

## Standard Path

- `M-01 old-belief-index`
- `M-02 expectation-break`
- `M-03 new-framework`
- `M-04 boundary-and-timing-anchor`
- `M-05 identity-embed`

## Required Fields Per Module

- id
- name
- K/P/A targets
- input state
- logic steps
- output state
- interface hook

## Practical Standard

Each module should have one dominant job.

Bad:

- M-01 already teaches the full framework
- M-03 starts doing emotional wrap-up

Good:

- M-01 names the old frame
- M-02 breaks it
- M-03 replaces it
- M-04 tells the reader how to orient in time or sequence
- M-05 makes the change easier to retain and repeat
