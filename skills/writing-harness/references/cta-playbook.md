# CTA Playbook

Use this when you want the harness to behave like a cognitive task analysis system rather than a generic editorial system.

CTA here means:

- extract expert decision structure
- surface hidden reader heuristics
- convert abstract explanation into executable cognitive scripts

## Core Rule

Do not merely explain what is true.

Extract how a strong expert would:

- recognize the situation
- notice the decisive signal
- reject the wrong default
- choose the next move
- rehearse the new judgment

## Module-by-Module CTA Load

### `M-01 old-belief-index`

Use:

- think-aloud simulation
- reader first-frame image
- named default heuristics

Required outputs:

- at least 3 reader inner-monologue lines or frame labels
- named default processes

### `M-02 expectation-break`

Use:

- critical incident
- absurd conclusion from the old tree
- expert switch parameter

Required outputs:

- one hard contradiction
- one explicit collapse point for the old logic

### `M-03 new-framework`

Use:

- GOMS-style decomposition
- executable method steps
- if-then rules

Required outputs:

- goal
- operations
- methods
- selection rules

### `M-04 boundary-and-timing-anchor`

Use:

- temporal cognition
- milestone rehearsal
- observable future checkpoints

Required outputs:

- now / soon / later timeline
- checklist or fill-in template
- one downgrade condition if reality does not confirm the thesis

### `M-05 identity-embed`

Use:

- cognitive rehearsal
- role-play
- rebuttal lines
- attribution locking

Required outputs:

- 3 retell bullets
- 3 rebuttal lines
- one identity sentence in the form:
  - `Although ..., I choose ...`

## Practical Test

If the output can be read passively but not used actively, the CTA layer has failed.
